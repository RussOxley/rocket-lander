Replace the personalWeight, skillPressure, and promoEdge logic:

Old:


const personalWeight = clamp(
  0.12 + 0.58 * clarity + 0.18 * Math.abs(skillMu - 0.5) * 2,
  0.12,
  0.86

// ...
const skillPressure = clamp(0.72 * clarity + 0.20 * Math.max(0, skillMu - 0.5) * 2, 0, 0.92);

New:


// Converges to your personal hit-rate faster once skill is clear (ceiling raised 0.86 → 0.96)
const personalWeight = clamp(
  0.10 + 0.72 * clarity + 0.26 * Math.abs(skillMu - 0.5) * 2,
  0.10,
  0.96
);
// ...
// skillSurcharge: market charges a spread once it's confident you're above-average.
// Kicks in at clarity > 0.40 + skillMu > 0.52. Can make promoEdge negative.
const skillPressure = clamp(0.88 * clarity + 0.38 * Math.max(0, skillMu - 0.5) * 2, 0, 1.3);
const skillSurcharge = clamp((clarity - 0.40) * Math.max(0, skillMu - 0.52) * 0.10, 0, 0.020);
const promoEdge = clamp(baseEdge * (1 - skillPressure) - skillSurcharge, -0.015, baseEdge);
Also add skillSurcharge to the returned object so the UI can display it:


return {
  qBook, qPersonal, qQuote, riskFreeRate,
  promoEdge, skillSurcharge,   // <-- add skillSurcharge here
  profitMult: quotedProfitMult(qQuote, riskFreeRate, promoEdge),
  personalWeight, clarity,
};
Why this works: A skilled player (high skillMu, high clarity) faces two headwinds simultaneously — their qQuote rises toward their true hit-rate (so profitMult compresses from above), AND the skillSurcharge pushes promoEdge negative (compresses from below). An overconfident player who keeps betting big despite these compressed margins is revealing exactly the overconfidence you want to detect.

const expectedBonusPct = effectiveFuelCap * expectedFuelLeftFrac(...);
And apply the same fuelSkillDiscount in resolveSuccess (line ~1066):


const fuelSkillDiscount = clamp(skillClarity(skillState.sigma2) * Math.max(0, skillState.mu - 0.5) * 1.2, 0, 0.7);

Player state	qQuote shift	promoEdge	Fuel bonus cap
